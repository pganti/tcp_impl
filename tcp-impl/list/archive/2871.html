    > Subject: URG in pure ACKs
    > Date: Wed, 25 Sep 2002 04:31:47 -0700
    > From: "Prakash Khemani" <khemani@netscaler.com>
    >
    > Is it true that the TCP sender MUST not (or is it SHOULD not) set the
    > URG data pointer in pure ACK packets? Is it a MUST (or SHOULD) that the
    > URG pointer can be set by the sender only if the packet carries at least
    > 1 byte of payload? Any RFC pointers?

    No, it is not true. There is nothing wrong with having the URG bit
    set in an ACK only packet. The urgent pointer is independent of any
    data in the packet; it is just an offset relative to the SEQ field and
    can point to an octet beyond the data in the current packet. (Though
    in reality, if the urgent pointer points beyond the data in the current
    packet, all it is really saying is "time to go into urgent mode, you'll
    get the actual urgent pointer in a later packet". You can read RFC 2147
    for the discussion of using the Urgent Pointer with IPv6 Jumbograms.)
