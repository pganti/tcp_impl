>
> But, unless we deploy something like RED, remember that there are results
> suggesting that TCP connections tend to synchronize in the bottleneck
> queue, with the result that the smallest TCP connection gets penalized more
> often than one would expect. So we may nail one connection until we've
> driven it to a tiny window, then start reducing the next connection.
>
> In any case, our behavior will, in no way, correctly approximate that of
> the N*W window size single TCP.
>

If the connections tend to synchronize, won't every connection experience a
loss upon switch buffer overflow ? This effect has already been demonstrated in
simulation studies of TCP dynamics. If all connections lose a packet, the
effect would be similar to that of a single connection with N*W window. The
only main difference would be that the rate of window increase with a single
connection is smaller than with multiple connections.

- Mohit
