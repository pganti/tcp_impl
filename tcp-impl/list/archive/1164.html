A TCP connection is defined by the unique combination of
source/destination IP address and source/destination TCP port.

So, between two specific machines (IP addresses), you have
a theoretical maximum of 2^16*2^16 simultaneous connections.
If all connections to to the same port, then you can have
at most 2^16 connections.

But usually you are limited to values below that by the OS
(like the maximum number of open file descriptors). In BSD
systems, if you don't explicitly set the port number with
bind(), the kernel will select one for you. Unless the vendor
has changed this, the range from which dynamic ports will be
assigned is 1024-5000, or just under 4000 connections.
