      *> Is it true that the TCP sender MUST not (or is it SHOULD not) set the
      *> URG data pointer in pure ACK packets? Is it a MUST (or SHOULD) that the
      *> URG pointer can be set by the sender only if the packet carries at least
      *> 1 byte of payload? Any RFC pointers?
      *>
      *> -- prakash
      *>

    It would seem that the urgent pointer should be set by node X if and
    only if X's end of the connection has unacknowledged urgent data to
    send, regardless of whether or not the packet being sent contains
    data. As an aside, if there is urgent data to be sent then the packet
    probably will contain data.
