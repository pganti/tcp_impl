> Note that all calls are unconditional (except for uninteresting cases
> like shutdown() on a non-socket). If the socket is already in
> FIN_WAIT_2, another shutdown()-for-sends will, as I read it, forcibly
> disconnect both directions (soisdisconnected) and start the 2MSL timer.

If the socket is already in FIN_WAIT_2, the application must have already
called close() or shutdown() on it. Calling shutdown() again certainly seems
to disconnect the socket but I don't think any reasonable application will
cause shutdown() twice on a socket.

Ideally speaking though, the second shutdown should be unsuccessful rather than
disconnecting the socket.
