On Fri, 24 Apr 1998, Vernon Schryver wrote:

> > From: "David S. Miller" <davem@dm.cobaltmicro.com>

> > Firstly, it is not a configurable parameter for most browsers I know
> > of, and can't be changed unless you have the source to the browser.
>
> [...]
> Moreover, since the proposed change to the initial window size is not
> configurable, you would need new HTTP PC client to implement it, and it
> seems reasonable that when (and if) your browser vendor changes the initial
> window size in the TCP stack it ships, it will also change the default
> number of TCP streams.
 
The draft HTTP/1.1 standard includes language to restrict the number of
simultaneous connections to be created by a conformant client to a given
server:
  
] Clients that use persistent connections SHOULD limit the number of
] simultaneous connections that they maintain to a given server. A
] single-user client SHOULD maintain AT MOST 2 connections with any
] server or proxy. A proxy SHOULD use up to 2*N connections to another
] server or proxy, where N is the number of simultaneously active users.
] These guidelines are intended to improve HTTP response times and avoid
] congestion of the Internet or other networks.

This is independant of any change to the underlying TCP, and it seems
likely that the browser applications will be 1.1 before the TCP layers
get changed in any event (IE4 is already 1.1).
