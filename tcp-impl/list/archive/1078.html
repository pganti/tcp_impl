In message <Pine.LNX.3.96.980423084412.22897B-100000@hill.ucr.edu>, Natchu Vish
nu Priya writes:

>and when the network is congested.. this minimum window size will
>effectively result in a increase in data in network till the rtt matches
>the available bandwidth and the window size of 4K...

No, it says very clearly the minimum window size is 1 segment.

>For a concrete example....
>
>I'm browsing over a 28.8 K modem link..
>
>with 3 sim. connections thats a combined 12K of data in flight..

You should never get 12K in flight -- your should have a small queue
size at the ingress to the modem link. So you'll get, say 4K in flight
and 8K dropped, and then fall back to 1K window size -- per the simulations
Tim Shepard and I did.
