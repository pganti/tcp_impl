    > From: Manish Jain <jain@cc.gatech.edu>
    > Subject: changing advertized receiver window
    >
    > Hi,
    > It seems that socket api (in solaris and bsd's) "does not" allow the
    > receive buffer in TCP
    > to be changed(increased/decreased) once the connection has been
    > established.
    ...
    > Does the TCP protocol require this behavior ? I read the rfc 793
    > and couldnt find anything in there which imposes such a restriction
    > on receiver.

    Shrinking the TCP window is a bad thing, so if you decrease the size
    of the receive socket buffer of an established connection, most TCP
    implementations will continue to advertise the old window. As new data
    is received and acknowledged, the left edge of the window will advance
    and the right edge will remain fixed until enough new data has been
    received to get the window down to the new size.

    If you increase the socket buffer size, most TCP implementations should
    reflect that when the next TCP window update is sent.
