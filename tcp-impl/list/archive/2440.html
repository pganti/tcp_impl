    > PPP/SLIP routers tend to slow down a bit with timestamps because they
    > cannot use VJ compression anymore and the header is transmitted fully.
    > Of course not the router really slows down, the latency of the link
    > just increases a bit because more bytes need to be transfered.
    > It is usually only noticeable with small packets and very slow links.

    As recently as this past december, I've run into (well, dialed into)
    PPP implementations "in the wild" which miscompressed (or, perhaps,
    misuncompressed) packets with RFC1323 timestamps... got horribly laggy
    performance until I turned them off on the laptop and then *poof* it
    started working as expected.
