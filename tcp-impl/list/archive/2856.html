    Hi,
    It seems that socket api (in solaris and bsd's) "does not" allow the
    receive buffer in TCP
    to be changed(increased/decreased) once the connection has been
    established.

    socket(...)
    setsockopt(.., SO_RCVBUF, ...)
    connect(...)
    setsockopt(..., SO_RCVBUF...)

    The second call to setsockopt doesnot have any effect on the
    receive buffer.
    Does the TCP protocol require this behavior ? I read the rfc 793
    and couldnt find anything in there which imposes such a restriction
    on receiver.
    Any comments.
