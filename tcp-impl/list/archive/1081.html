Mark Allman wrote:

> N connections with an initial window
> of 1 segment are N times more aggressive throughout the transfer.

And I got wondering -- is that really true? I think the answer is more
complex. During the initial slow start, clearly we've got N connections
doubling their window in parallel. But in other cases, the situation is
more complex.

In particular, consider a case where all N connections are sending at their
full window size, W, and compare their congestion response with a single
connection sending with window size W*N.

If a router drops a datagram to signal congestion, the single connection
will do a fast recovery and resume sending with window size of W(N/2).
With N connections, one connection will do a fast recover and resume sending
with window size W/2 -- so the N connections have an effective window of
W((2N-1)/2) or nearly twice as large as the single connection.

We can continue this logic a bit further. Suppose that after a little while
(and RTT or two) the link is still congested and we drop another datagram.
The single connection will scale back to W(N/4). The N connections will do
one of two things. If the dropped datagram is for a different connection
from the original one, we'll shrink the window to W(N-1) (i.e. we're now
roughly 4 times bigger than the single connection). If the dropped datagram
is for the same connection as before we'll shrink to W((4N-3)/4).

Note that, in this case, we'd like all N connections to reduce their
windows, but instead we're getting just one -- with the result that we're
getting nothing close to exponential reductions.

Craig
