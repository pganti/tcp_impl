Alan Cox writes:
> > So what is that reason? Connections through a stateful firewall that
> > drops state if it sees no traffic for a relatively short period. (I
> > think the firewall in question has its timeout set to 5 minutes.) I
> > used to lose connecctions regularly until I realized what was going on
> > and hacked run-time tuning for those keepalive values into the kernel I
> > was using.
>
> "I'd like to jam the internet because my firewall is faulty". Thats the
> same mentality as the linear backoff brigade.

Pardon my flame, but, *what*?????

You can argue that keepalives cause all sorts of problems, but a short
packet every two hours on idle connections is going to jam the
internet? Especially when most connections aren't set for keepalives?
Get real. You'd need a probe of some sort anyway if you don't want to
be obnoxious and throw off legitimately idle connections.
